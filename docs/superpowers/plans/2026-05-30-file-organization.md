# File Organization Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** ทำ root directory สะอาด, gitignore ครอบคลุม, และจัด docs/ ให้มีโครงสร้างชัดเจน

**Architecture:** แก้ไขไฟล์ `.gitignore` 2 จุด (root + Frontend), ใช้ `git rm --cached` เพื่อ untrack uploads ที่ติด git อยู่, ย้าย `planning.md` ไป `docs/`, และสร้าง `docs/README.md` + `docs/reports/`

**Tech Stack:** Git, Markdown

---

- Root `.gitignore` มี `.DS_Store`, `Thumbs.db`, และ `planning.md` อยู่แล้ว — จะลบ `planning.md` rule ออก (เพราะไฟล์ย้ายไป docs/) และเพิ่ม `*.pdf`, `testcases_*.txt`, `desktop.ini`
- Working directory: `C:\xampp\htdocs\Co_project`
